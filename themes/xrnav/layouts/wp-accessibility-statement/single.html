Accessibility Statement for XR Navigation and all its applications

This is an accessibility statement from XR Navigation Inc.

Here are the Accessibility Conformance Reports for all XR Navigation products and domains.

Measures to support accessibility

XR Navigation Inc. takes the following measures to ensure accessibility of all its applications:

  • Include accessibility as part of our mission statement.
  • Include accessibility throughout our internal policies.
  • Integrate accessibility into our procurement practices.
  • Provide continual accessibility training for our staff.
  • Assign clear accessibility goals and responsibilities.
  • Employ formal accessibility quality assurance methods.
  • Employ disabled developers.
  • Have people with disabilities as executives and leaders within the company.
  • Perform frequent user testing with different groups of users.
  • Employ IAAP certified developers as core members of the design and development process.
  • Integrate accessibility and user experience into the same conversations.
  • Obtain independent accessibility audits for products and sites.

Conformance status

The Web Content Accessibility Guidelines (WCAG) defines requirements for designers and developers to improve accessibility for people with disabilities. It defines three levels of conformance: Level A, Level AA, and Level AAA. all XR Navigation applications are fully conformant with WCAG 2.1 level AA (although we strive for AAA). Fully conformant means that the content fully conforms to the accessibility standard without any exceptions.

Feedback

We welcome your feedback on the accessibility of all XR Navigation applications. Please let us know if you encounter accessibility barriers on any XR Navigation application:

  • E-mail: support@xrnavigation.io
  • https://xrnavigation.io/contact

We try to respond to feedback within 48 hours.

Compatibility with browsers and assistive technology

All XR Navigation applications are designed to be compatible with the following assistive technologies:

  • Windows on Chrome, Edge, and Firefox with NVDA
  • Windows on Chrome, Edge, and Firefox with Jaws
  • Mac on Safari, Chrome, and Firefox with Voice Over
  • iOS on Safari, Chrome, and Firefox with Voice Over

XR Navigation applications are probably not compatible with:

  • Internet Explorer
  • Any browser that does not support the Web Audio API
  • Environments and screen readers that do not support Aria and JavaScript

Technical specifications

The accessibility of all the XR Navigation applications rely on the following technologies to work with the particular combination of web browser and any assistive technologies or plugins installed on your computer:

  • HTML
  • WAI-ARIA
  • CSS
  • JavaScript

These technologies are relied upon for conformance with the accessibility standards used.

Limitations and alternatives

Despite our best efforts to ensure accessibility of all the XR Navigation applications , there may be some limitations. Below is a description of known limitations, and potential solutions. Please contact us if you observe an issue not listed below.

Known limitations for all XR Navigation applications:

  1. If you’re blind, Non-Visual Desktop Access (NVDA) works best with all our products because it is the screen reader our development team primarily uses. Sometimes other screen readers, such as Jaws or Voice Over, may not have an optimal experience because Each screen reader has slightly different implementations of web accessibility features e.g., aria live regions. We are constantly working with screen reader developers and accessibility specification writers to help make the experience uniform across different screen readers, and we provide workarounds whenever possible. Note that NVDA is a free screen reader on Windows, and sometimes the solution is to use it instead of other screen readers. Please contact support@xrnavigation.io if you find any issues, and note the best experience is with NVDA.
